# AI Usage Report

Throughout Sprint 2, our team continued to leverage Generative AI tools (such as ChatGPT, Gemini, and Copilot) to accelerate development, improve code quality, and maintain clear documentation. As the project evolved from basic exploration to deeper feature implementation, our use of AI shifted towards debugging complex issues, refactoring, and preparing for testing. 

### 1. Accelerating Advanced Development & Testing
With the foundational architecture set in Sprint 1, we tackled more complex features this sprint. We used AI as a pairing partner to speed up repetitive coding tasks and optimize our logic. 

* **Refactoring and Optimization:** We asked AI to suggest cleaner, more efficient ways to write certain React Native components and Spring Boot services, helping us reduce technical debt early on.
* **Complex Debugging:** When facing hard-to-track bugs—especially concerning asynchronous operations or state management—we used AI to analyze error logs and pinpoint potential edge cases we might have missed.
* **Generating Test Data & Boilerplate:** To thoroughly test our new features, we used AI to quickly generate realistic mock data (JSON payloads) and draft the initial boilerplate for unit tests, saving us valuable manual entry time.

### 2. Maintaining & Updating Documentation
As the app's functionality grew, keeping the documentation up to date became crucial. We relied on AI to ensure our technical writing remained accurate and accessible.

* **API Documentation:** We used AI to help structure and format our API endpoint documentation, ensuring the contracts between our frontend and backend were clearly defined and easy to read.
* **Updating Existing Specs:** We fed our updated implementation details to the AI to help us quickly revise the technical documentation created in the previous phases, ensuring consistency.

### 3. Workflow & Code Review Assistance
To maintain a high standard of code quality without slowing down our development velocity, we integrated AI into our review processes.

* **Pull Request Summaries:** We occasionally used AI to help draft clear and concise descriptions for our Pull Requests, making it easier for team members to understand the changes before reviewing the code.
* **Spotting Edge Cases:** Before finalizing a feature, we would prompt the AI to suggest potential edge cases or missing validations, acting as a "second set of eyes" to strengthen our logic.
