# Pilot User Testing - Sprint 2

Simple validation process with 2 groups of regular app users.

## Documents

- **[Strategy.md](Strategy.md)** → Full plan: groups, timeline, how we'll react to feedback, GitHub issue workflow
- **[Feedback_Form.docx](Feedback_Form.docx)** → Questionnaire for testers to report bugs/feedback

## Pilot Groups

- **Group 1:** 3-4 users (1h testing)
- **Group 2:** 2-3 users (1h testing)

Both groups: Regular app users only. No code access.

## What we're testing

9 core use cases:
- US-01 (Registration), US-03 (Login), US-04 (Edit Profile)
- UC-01 (Location Questions), UC-02 (Voting), UC-04 (Event Forum), UC-06 (Business Events)
- UC-03 (Rewards - P1)

## Quick start

### For pilot coordinator (Darío Zafra Ruiz)
1. Prepare APK/ESC build
2. Create test accounts
3. Send app + credentials + questionnaire to testers
4. Collect feedback via email for 2-3 days
5. Triage findings → Create GitHub issues

### For testers
1. Download app + install
2. Use test account to log in
3. Test app normally (1h)
4. Fill out questionnaire: bugs found + severity
5. Email completed form back to coordinator

##  Feedback process

**During Testing:**
- Testers use app, note any bugs/issues

**After Testing (Days 1-3):**
- Testers submit DOCX questionnaire via email
- Report bugs: 🔴 CRITICAL / 🟠 MAJOR / 🟡 MINOR

**Team Reaction (Days 3-5):**
- Consolidate all feedback
- Create GitHub issues per finding
- Triage meeting: prioritize fixes for Sprint 2
- Dev team fixes critical/major items
